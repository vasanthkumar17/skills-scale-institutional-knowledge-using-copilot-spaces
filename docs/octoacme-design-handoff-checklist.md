# OctoAcme Design Handoff Checklist

## Purpose
Provide a reusable checklist to ensure that designs are complete, reviewed, and ready for development before implementation begins. Facilitates a smooth handoff between UX Designers, Developers, and Product Managers.

## Roles Referenced
| Abbreviation | Role |
|---|---|
| UX | UX Designer |
| Dev | Developers |
| PdM | Product Manager |
| QA | QA Engineer |

See [Roles and Personas](octoacme-roles-and-personas.md) for full descriptions.

---

## Design Completeness (Owner: UX Designer)
- [ ] All screens and states designed (default, empty, error, loading, success)
- [ ] Responsive/adaptive breakpoints defined (if applicable)
- [ ] Accessibility requirements specified (color contrast, ARIA roles, keyboard navigation)
- [ ] Design tokens and component library references documented
- [ ] Interaction and animation details annotated

## Design Review (Owner: UX Designer, Reviewed by: PdM, Dev)
- [ ] Designs reviewed and approved by Product Manager (PdM)
- [ ] Feasibility review completed with Developers — no blocking technical constraints
- [ ] Open feedback from review addressed and designs finalized
- [ ] Edge cases from Business Analyst or QA incorporated

## Handoff Assets (Owner: UX Designer)
- [ ] Design files shared in agreed tool (e.g., Figma, link in project repo)
- [ ] Exported assets (icons, images) provided in required formats and resolutions
- [ ] Design specs (spacing, typography, colors) exported or linked
- [ ] Clickable prototype available for reference (if applicable)

## Developer Ready (Owner: Developers, Confirmed with: UX Designer)
- [ ] Developers have access to design files and assets
- [ ] Design annotations and specs reviewed by implementing Developers
- [ ] Questions and clarifications raised and resolved before sprint start
- [ ] Acceptance criteria updated to include UX/visual requirements

## QA Alignment (Owner: QA Engineer, Consulted: UX Designer)
- [ ] Visual and interaction acceptance criteria captured in test cases
- [ ] Accessibility test cases written (if applicable)
- [ ] Design reference available to QA for implementation validation
